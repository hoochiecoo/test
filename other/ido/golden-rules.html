✨ Золотые правила интерфейса Бена Шнейдермана

1. Стремитесь к согласованности

Последовательные последовательности действий, одинаковое использование терминологии в подсказках, меню и экранах справки, одинаковые соглашения в команде программистов.

  • Одинаковые команды выполняют одинаковые действия везде
  • Используются единообразные символы, значки и цвета
  • Терминология согласована на всех экранах
  • Стиль диалогов одинаков везде
  • Расположение элементов управления предсказуемо
  • Одинаковые сообщения для одинаковых ситуаций
  • Процедуры одинаковые для похожих задач
2. Позволяйте частое пользователям инициировать действия

Пользователи должны иметь возможность инициировать действия, а не просто реагировать. Дайте пользователям возможность выбирать, что делать, а не только следовать предопределенным путям.

  • Пользователи контролируют ход диалога
  • Нет обязательной последовательности шагов
  • Можно переходить между функциями по выбору
  • Предоставляются быстрые клавиши для опытных пользователей
  • Пользователь может остановить долгие операции
  • Есть опции для пропуска или ускорения процесса
3. Предлагайте информативные обратные связи

Для каждого пользовательского действия должна быть система обратной связи. Срочность обратной связи должна соответствовать величине действия.

  • Система немедленно реагирует на все действия
  • Статус всегда видим пользователю
  • Долгие операции показывают прогресс
  • Ошибки сообщаются четко и понятно
  • Успешные действия подтверждаются
  • Сообщения об ошибках предлагают решения
  • Тон обратной связи позитивный
4. Разработайте диалоги для завершения

Диалоги должны информировать пользователей об их статусе в завершении действия. Группируйте действия логически, начиная с простых операций и переходя к сложным.

  • Четко определены начало, середина и конец действия
  • Пользователь знает, где он находится в процессе
  • Последовательность действий логична и интуитивна
  • Есть индикаторы прогресса для многошаговых процессов
  • Можно вернуться к предыдущим шагам
  • Завершение четко обозначено
  • Опционально - возможность сохранения прогресса
5. Предотвращайте ошибки

Разработайте систему так, чтобы пользователи не могли совершить серьезных ошибок. Если проблема возникает, система должна предложить четкие, конструктивные, простые инструкции по исправлению.

  • Система предотвращает опасные действия
  • Используется валидация ввода данных
  • Деструктивные действия требуют подтверждения
  • Система дает предупреждения перед ошибкой
  • Предлагаются ограничения на допустимые действия
  • Есть возможность отмены действий
  • Критические функции дополнительно защищены
6. Предоставляйте простые выходы из ошибок

Пользователи часто выбирают неправильные функции. Предусмотрите четко обозначенные выходы, не требующие обширного диалога. Поддерживайте функции "отмена" и "повтор".

  • Ясно, как вернуться на предыдущий экран
  • Есть кнопка "Отмена" на всех диалогах
  • Поддерживается функция Undo для отмены действий
  • Есть функция Redo для повтора действий
  • Выход из приложения просто и безопасно
  • Никаких скрытых ловушек и зависаний
  • Можно легко вернуться в главное меню
7. Поддерживайте внутренний локус контроля

Опытные пользователи сильно хотят чувствовать себя ответственными за систему и видеть, что они контролируют то, что происходит. Избегайте режимов, которые вводят пользователей в "режим модальности" без их запроса.

  • Пользователь чувствует себя ответственным за действия
  • Система не совершает неожиданные действия
  • Нет модальных диалогов, которые мешают работе
  • Пользователь может отменить любое действие
  • Система не принимает решения за пользователя
  • Пользователь может выбирать опции и параметры
  • Есть информация о том, что система будет делать
8. Сокращайте нагрузку на рабочую память

Человек может удерживать в памяти только ограниченное количество информации. Интерфейсы должны быть визуально простыми, задачи разбиты на подзадачи и наведена справочная информация по коротким инструкциям.

  • Экран содержит только необходимую информацию
  • Сложные задачи разбиты на простые шаги
  • Ненужная информация скрыта
  • Часто используемые опции видны и доступны
  • Нет необходимости запоминать информацию между экранами
  • Есть справка прямо на экране
  • Меню краткие и логически организованы
Восемь золотых правил дизайна интерфейса

Эти восемь правил Бена Шнейдермана служат основой для создания высокоэффективных и удобных для пользователя интерфейсов, которые прощают ошибки, позволяют пользователям легко восстанавливаться после ошибок и сводят к минимуму необходимость запоминания.

  • Последовательность через все операции
  • Пользователи инициируют все действия
  • Информативная и своевременная обратная связь
  • Замыкание задач и действий
  • Предотвращение ошибок перед выполнением
  • Простые и понятные сообщения об ошибках
  • Возможность отката без расширенных диалогов
  • Минимальная нагрузка на память пользователя
