🚀 Flutter Best Practices

Настройка окружения Flutter

  • Версия Flutter SDK задокументирована и заблокирована
  • Совместимая версия Dart SDK указана
  • Все члены команды используют одинаковую версию Flutter
  • Git настроен для игнорирования артефактов сборки
  • .pubignore настроен правильно
  • Flutter channel stable используется для продакшена
  • Переменные окружения настроены

Качество Кода и Архитектура

  • Следовать принципам чистого кода
  • Использовать решение управления состоянием (Provider, Riverpod, Bloc)
  • Отделить бизнес-логику от UI
  • Использовать значимые имена классов и методов
  • Избегать глубоко вложенных виджетов
  • Извлечь переиспользуемые виджеты
  • Правильная обработка ошибок и управление исключениями
  • Отсутствие жестко кодированных строк (использовать локализацию)
  • Константы определены в отдельном файле

Тестирование

  • Модульные тесты для всей бизнес-логики (>80% покрытие)
  • Тесты виджетов для компонентов UI
  • Интеграционные тесты для критических потоков
  • Запускать тесты перед каждым коммитом
  • Мокировать внешние зависимости
  • Тестировать сценарии ошибок
  • Тесты производительности для критических путей
  • Тестирование на нескольких размерах устройств

Управление Зависимостями

  • Минимизировать внешние зависимости
  • Использовать только активно поддерживаемые пакеты
  • Держать файл pubspec.lock в контроле версий
  • Регулярно обновлять зависимости
  • Проверять журналы изменений перед обновлениями
  • Тестировать после каждого обновления зависимости
  • Правильно использовать ограничения версий (^, ~)
  • Задокументировать причину критических зависимостей

Оптимизация Производительности

  • Использовать const конструкторы где применимо
  • Внедрить стратегии кэширования виджетов
  • Оптимизировать отрисовку списков (ListView.builder)
  • Использовать RepaintBoundary для сложных виджетов
  • Профилировать приложение с DevTools
  • Мониторить использование памяти и предотвращать утечки
  • Оптимизировать загрузку и кэширование изображений
  • Использовать ленивую инициализацию для тяжелых операций
  • Внедрить правильное сохранение состояния

Конфигурация Сборки

  • Отдельные конфигурации сборки для dev/staging/prod
  • Различные ID приложений для разных сборок
  • Специфичные для окружения эндпоинты API
  • Различные иконки приложения для разных сборок
  • Правильно настроены сертификаты подписи
  • Release сборки оптимизированы (--split-per-abi)
  • Правила ProGuard/R8 настроены для Android
  • Bitcode включен для iOS

iOS Специфика

  • Целевая версия iOS задана правильно (12.0+)
  • Podfile обновлен и заблокирован
  • Сертификат подписи и профиль provisioning действительны
  • Подписание приложения настроено автоматически или вручную
  • Политика конфиденциальности и деклараций данных добавлены
  • Протестировано на iPhone и iPad
  • Обработка безопасной зоны и notch протестирована
  • Поддержка темного режима проверена
  • Протестированы функции, специфичные для версии iOS

Android Специфика

  • minSdkVersion установлена правильно (API 21+)
  • targetSdkVersion установлена на последнюю
  • Android manifest настроен правильно
  • Необходимые разрешения объявлены
  • Файл keystore защищен резервной копией
  • Ключ подписи приложения настроен
  • Протестировано на различных версиях Android
  • Протестировано на телефонах и планшетах
  • Поведение кнопки "Назад" правильное

Развертывание в App Store и Play Store

  • Название приложения и описание завершены
  • Иконки приложения и скриншоты подготовлены
  • Политика конфиденциальности проверена и обновлена
  • Номер версии и номер сборки установлены правильно
  • Примечания к выпуску подготовлены
  • Анкета оценки содержимого заполнена
  • Ключевые слова и оптимизация поиска готовы
  • Переводы скриншотов и превью готовы
  • Сборка протестирована через TestFlight/Internal Testing

Мониторинг и Аналитика

  • Firebase настроена и интегрирована
  • Отчеты о сбоях включены
  • События аналитики отслеживаются
  • Мониторинг производительности активен
  • Реализовано пользовательское логирование ошибок
  • Отслеживание сеанса пользователя включено
  • Панель мониторинга в реальном времени настроена
  • Оповещения настроены для критических проблем
