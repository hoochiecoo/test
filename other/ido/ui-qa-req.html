🛡️ UI/QA Requirements

Фокус: минимизация ошибок пользователя, предотвращение неверного ввода и снижение когнитивной нагрузки. Это не про стиль — это про функциональную надежность, которая предотвращает потерю данных, финансовые ошибки и пользовательское разочарование.
✅ Группа 1: Валидация и контроль ввода
🔒Блокировка кнопки (Disable until valid)
Кнопка отправки остается неактивной, пока все обязательные поля не заполнены корректно.
Визуальная индикация: серая кнопка, не реагирует на наведение
Включает реал-тайм проверку всех required полей
Срабатывает при: пустое поле, неверный формат, данные вне диапазона
Дополнение: tooltip при наведении объясняет, что не хватает ("Заполните Email")
✅ Предотвращает отправку битых данных и лишние срабатывания валидации
🎭Маски ввода (Poka-Yoke)
Автоматическое форматирование данных в процессе набора. Система физически не дает ввести буквы там, где нужны цифры.
Номер карты: 4444 4444 4444 4444 (пробелы автоматически)
Телефон: +7 (999) 123-45-67 (скобки и дефисы автоматически)
Дата: ДД.ММ.ГГГГ (слэши добавляются после ввода цифр)
Отклонение символов: при попытке ввести букву в цифровое поле — звуковой/визуальный feedback
СНИЛС, ИНН, паспорт — специальные маски для России
✅ Исключает опечатки и форматирует данные правильно с первого раза
⚡Инлайновая валидация (Inline Validation)
Проверка поля сразу после того, как пользователь закончил ввод (onBlur), а не после нажатия кнопки.
Email: проверка при табе на другое поле (не на Enter)
Визуал: красная граница + значок ошибки + текст ошибки под полем
Зеленая граница и галочка при успешной валидации
Задержка 500мс перед проверкой (не валидирует при каждом символе)
Async валидация: проверка на сервере (есть ли email в БД), но с визуальным спиннером
✅ Исправляет ошибку в контексте, пока человек еще помнит, что вводил
💬Визуальные подсказки формата (Input Hinting)
Пример правильного заполнения внутри поля или под ним для предотвращения путаницы.
Placeholder: "Пример: name@example.com" (исчезает при фокусе)
Helper text под полем: "Используйте формат ДД.ММ.ГГГГ (29.12.2026)"
Иконки: календарь для дат, @-знак для email, скрепка для файлов
Длина ввода: счетчик символов для текстареи ("0 / 500 символов")
Примеры вариантов: "Например: Россия, США, Франция"
✅ Показывает ожидаемый формат, предотвращая путаницу
🔍Автодополнение и живой поиск (Autocomplete)
Предложение существующих вариантов прямо в строке ввода при наборе текста.
Адреса: при вводе "Мос" появляется список городов (Москва, Мосальск)
Email: при вводе "g" появляются @gmail.com, @google.com, @gmai.ru
Артикулы товаров: фильтрация по типу + цена + фото в dropdown
История поиска: последние 5 поисков пользователя
Fuzzy matching: поиск по неполному совпадению (поиск "Казань" по "Кзн")
Клик на вариант — автоматическая вставка и переход на следующее поле
✅ Исключает опечатки и ускоряет заполнение формы
📝Человекочитаемые сообщения об ошибках
Конкретные, понятные сообщения вместо технических кодов ошибок.
❌ ПЛОХО: "Error 404" или "Invalid input"
✅ ХОРОШО: "В пароле должна быть хотя бы одна заглавная буква и цифра"
Специфичность: указать точное поле ("Email" вместо "Data validation error")
Конструктив: не просто "Ошибка", а "Что делать дальше"
Тон: человеческий, без сложного жаргона ("Введите корректный номер" вместо "Invalid numeric input")
Ссылка на справку: для сложных ошибок добавить "Подробнее →"
✅ Дает четкую инструкцию, как исправить ошибку
⚠️ Группа 2: Предотвращение случайных и разрушительных действий
🚫Немедленная блокировка при клике (Prevent Double Action)
После нажатия кнопка мгновенно блокируется и меняет состояние на «Загрузка...». Исключает двойной клик.
Кнопка становится disabled после первого клика (невозможно нажать снова)
Визуал: серый цвет + спиннер + текст "Отправка..." или "Загрузка..."
Timeout: если ответ сервера > 60 сек, показать "Операция заняла долго, повторить?"
После завершения: restore кнопку в нормальное состояние или скрыть (если успех)
На мобильных особенно важна минимальная задержка и четкий feedback
✅ Исключает повторные списания денег или дублирование заявок в БД
↩️Отмена вместо подтверждения (Undo over Confirm)
Действие совершается мгновенно, но появляется кнопка «Отменить». Не прерывает поток работы.
Удаление письма: исчезает из папки, но внизу экрана toast "Письмо удалено" + кнопка "Отменить"
Timeout на отмену: 5-7 секунд, затем письмо удаляется окончательно
Перемещение задачи между досками: drag-drop, тут же toast с Undo
Удаление из корзины: вместо диалога "Вы уверены?" — сразу удали, но дай отменить
Batch операции: удалили 10 файлов, можешь отменить одну операцию на всех
Можно использовать Ctrl+Z (Win) / Cmd+Z (Mac) для отмены в веб-приложении
✅ Исправляет случайные клики, не прерывая поток работы
🔴Подтверждение разрушительных действий (Destructive Action Check)
Для фатальных операций требуется дополнительное подтверждение (ввод текста или капча).
Удаление аккаунта: требуется ввести слово "УДАЛИТЬ" или "Я понимаю риск"
Очистка всех данных: ввод пароля или двухфакторная проверка
Удаление проекта: ввод названия проекта ("Вы удаляете проект: [название]")
Восстановление БД: только после double-check и интервала 24 часа
Визуал: красная кнопка с предупреждением ⚠️, диалог модальный, нельзя отмахнуться
Email-подтверждение: ссылка из письма для экстра защиты
✅ Исключает случайную потерю данных из-за одного неверного клика
💾Защита от потери данных при выходе
Предупреждение при попытке закрыть страницу с незаполненной, но неотправленной формой.
Триггер: если в форме есть введенные (но не отправленные) данные
Диалог: "У вас есть несохраненные изменения" + кнопки "Остаться" / "Уйти все равно"
Для всех браузеров: window.onbeforeunload = подсказка
Не срабатывает, если форма пуста или уже отправлена
На мобильных: extra careful — может быть аккциденталь close вкладки
✅ Предотвращает потерю работы при случайном закрытии вкладки
🧠 Группа 3: Упрощение и снижение когнитивной нагрузки
🎯Умные значения по умолчанию (Smart Defaults)
Система заранее выбирает самые вероятные варианты, уменьшая объем ручной работы.
Страна по IP: пользователь из РФ → "Россия" предвыбрана
Текущая дата: при создании события дата по умолчанию = сегодня
Валюта: USD для США, RUB для России, EUR для Европы
Язык: определить по браузеру или cookie пользователя
Последние используемые: адрес доставки (если заказывал раньше)
Правильный часовой пояс: автоматически из системы
✅ Чем меньше полей юзер трогает вручную, тем меньше шансов на ошибку
📂Прогрессивное раскрытие (Progressive Disclosure)
Скрытие сложных или опасных настроек под кнопку «Дополнительно» или расширяемый раздел.
Форма подписки: базовые поля (Email, Пароль), а "Две фактор-аутентификация" под "+ Дополнительно"
Настройки: новичок видит 5 базовых опций, опытный нажимает "Расширенные" и видит 30
Безопасность: удаление аккаунта, API ключи, ограничение IP — в отдельной вкладке
Визуал: кнопка "+ Еще" или стрелка ▼ "Показать детали"
Новичок не сможет случайно изменить критический параметр, что он не понимает
✅ Интерфейс остается простым, но сложность доступна при необходимости
🪜Пошаговые мастера (Wizards)
Разбиение сложной формы на несколько простых этапов с прогрессом.
Форма из 20 полей → 4 простых этапа (Шаг 1/4, 2/4, 3/4, 4/4)
Progress bar вверху показывает прогресс (50% заполнено)
Кнопки: "← Назад", "Далее →" (или "Отправить" на последнем шаге)
Валидация: каждый шаг проверяется перед переходом на следующий
Сохранение прогресса: можно вернуться на шаг 2 и изменить данные
Визуал: стандартная структура (числа 1, 2, 3 с галочками для завершенных)
✅ Снижает риск запутаться в логике, пользователь сконцентрирован на одном этапе
🥖Хлебные крошки (Breadcrumbs)
Постоянная индикация пути в иерархии, помогает понять текущее положение.
E-shop: Главная > Электроника > Ноутбуки > ASUS Vivobook 15
Админ-панель: Dashboard > Пользователи > ID:12345 > Действия
Интерактивность: кликабельные links — один клик вернуться на "Электроника"
Последний элемент: текст (не ссылка), показывает текущую страницу
Разделитель: ">" или "/" или стрелка "→"
На мобильных: может быть компактнее или скрыто в меню
✅ Исключает дезориентацию, пользователь всегда знает, где он и куда может вернуться
🎚️Лимиты и ограничения (Constraints)
Использование слайдеров, выпадающих списков вместо свободного ввода текста где диапазон ограничен.
Дата: не текстовое поле, а календарь (невозможно ввести "32 февраля")
Количество товаров: слайдер от 1 до 100 (невозможно выбрать -5)
Выбор категории: dropdown с фиксированным списком (невозможно написать неверную категорию)
Диапазон цен: dual-range слайдер (от 1000 до 100000 руб)
Статус: radio buttons или segmented control (вкл/выкл, активен/неактивен)
Rating: звезды 1-5 (невозможно поставить 7 звезд)
✅ Физически невозможно выбрать неверное значение
👀Сводка перед подтверждением (Review Step)
Финальный экран со всеми введенными данными перед окончательной покупкой или отправкой.
E-commerce: сводка товаров, адрес доставки, способ оплаты, итоговая сумма
Авиабилеты: вылет, прилет, пассажиры, цена — все видно до оплаты
Формы: все поля отображены readonly для проверки (или edit ссылка для исправления)
Подсчет: сумма, налоги, скидки — все детали разобраны
Edit links: "Изменить адрес", "Изменить способ оплаты" — быстрый возврат без заново заполнения
Финальная кнопка: "Подтвердить заказ" (только после проверки)
✅ Позволяет увидеть и исправить опечатки в последний момент перед отправкой
🔐 Группа 4: Безопасность и удобство доступа
👁️Показать/Скрыть пароль (Password Toggle)
Иконка 👁️/👁️‍🗨️ позволяет проверить введенный текст, вдвое снижая ошибки при регистрации.
Иконка справа от поля: при клике текст показывается, при клике снова — скрывается
Состояние: • • • • • • (скрыто) → password123 (видно)
Избавляет от поля "Повторите пароль" (пользователь может сразу проверить)
На мобильных: иконка достаточно большая (минимум 44x44px для удобства)
При скрытии пароля: курсор остается в поле (не теряет фокус)
UX: по умолчанию пароль скрыт, toggle всегда доступна
✅ Вдвое снижает количество ошибок при регистрации и входе
✨Магическая ссылка (Magic Link)
Вход без пароля. Полностью устраняет ошибку "неверный пароль" или "я забыл пароль".
Вход: пользователь вводит только Email (без пароля)
Письмо: на указанный email приходит ссылка "Войти в аккаунт"
Ссылка валидна: 15-30 минут (не бесконечно)
Первый вход: если аккаунта нет, волшебная ссылка регистрирует пользователя
Опцион: пользователь может установить пароль позже (не обязательно)
Альтернатива OAuth: Google, GitHub, Apple для быстрого входа
✅ Полностью устраняет самую частую причину ухода пользователей (забыли пароль)
🤖 Группа 5: Контекстное понимание и автосохранение
🧠Контекстное распознавание ошибок (Context Awareness)
Система анализирует намерения и предупреждает о потенциальных ошибках перед их совершением.
Email: в письме есть слово "прикрепил", но файла нет → предупреждение перед отправкой
Платеж: выбрана валюта USD, но сумма 50,000 (может быть ошибка) → подтверждение
Спелчек: обнаружено много опечаток в письме → "Хотите исправить?"
Дата: выбрана дата в прошлом → "Вы уверены? Дата 2020 года"
Duplicate prevention: попытка добавить товар, который уже в корзине → "Уже добавлен"
✅ Перехватывает частые ошибки до их совершения
💾Сохранение черновиков (Autosave)
Автоматическое сохранение введенных данных каждые несколько секунд в локальное хранилище.
Интервал: сохранение каждые 5-10 секунд (не постоянно, чтобы не слаговать)
Хранение: localStorage (браузер) + серверное сохранение (опционально)
Визуал: маленький текст "Автосохранено" вверху, исчезает через 2 сек
При перезагрузке страницы: "Восстановить черновик? Сохранено 10 мин назад"
Для текстареи, комментариев, длинных форм особенно важно
Не сохранять пароли и чувствительные данные в localStorage
✅ Предотвращает потерю работы при закрытии вкладки, сбое интернета или браузера
